# CV32E40P RTL-to-GDS Implementation Plan

## Goal

OpenHW `CV32E40P` RISC-V core를 대상으로 **Front-End부터 Back-End까지 연결되는 RTL-to-GDS flow**를 구축한다.

핵심은 기존 Ibex synthesis/STA baseline 반복이 아니다.

```text
GitHub RTL clone
→ source/filelist/include 정리
→ DC synthesis
→ Formality RTL-to-netlist
→ ICC2/OpenROAD physical implementation
→ post-route STA
→ PPA/timing/physical issue 분석
→ portfolio report
```

## Why CV32E40P

기존 프로젝트와 겹침 최소화:

```text
AES128 = 직접 설계 crypto SoC + synthesis/DFT/ATPG/STA/FM
Ibex = RISC-V synthesis/STA baseline + compile/compile_ultra + period sweep
ORCA = 끊어진 front-end flow reconstruction + DFT/ATPG/FM/STA debug
AXI-QSPI/IIC = AXI peripheral/FPGA integration

CV32E40P 신규 = 외부 산업형 RISC-V CPU IP의 RTL-to-GDS physical implementation
```

CV32E40P 선택 이유:

```text
OpenHW CORE-V 계열
32-bit in-order 4-stage RISC-V core
RV32IMC / RV32IMFC / Xpulp 지원
ASIC target 문서 존재
Ibex보다 산업용 IP 느낌 강함
CVA6/BOOM/NVDLA보다 현실적 크기
```

## Scope Decision

### Main config

처음은 작게 시작.

```text
Top: cv32e40p_top
FPU = 0
COREV_PULP = 0 initially
COREV_CLUSTER = 0
NUM_MHPMCOUNTERS = 1 or 0 if allowed
```

목표:

```text
RV32IMC-class core physical implementation baseline
```

FPU/PULP는 확장 only.

### Do not do initially

```text
FPU enabled full flow
CVA6/BOOM급 큰 core
Linux-capable SoC
full verification claim
ISA compliance claim
```

## Tool Strategy

Primary commercial flow:


## Phase 2 — Clock Gate Replacement

Goal: synthesis 가능한 `cv32e40p_clock_gate` 제공.

Problem:

```text
CV32E40P는 clock gate cell 필요
simulation clock gate는 synthesis intended 아님
```

Options:

### Option A — library ICG cell wrapper

권장.

```systemverilog
module cv32e40p_clock_gate (
  input  logic clk_i,
  input  logic en_i,
  input  logic scan_cg_en_i,
  output logic clk_o
);
  // instantiate technology ICG cell
endmodule
```

Map:

```text
en_i OR scan_cg_en_i → ICG enable/test enable
```

### Option B — Synopsys integrated clock gating inference

가능하지만 wrapper/interface 매칭 필요.

### Option C — latch-and-and generic RTL

프로토타입 only. final claim 약함.

Exit criteria:

```text
DC elaborate/link에서 unresolved cv32e40p_clock_gate 없음
scan_cg_en_i test behavior 설명 가능
```

## Phase 3 — DC Synthesis Baseline

Goal: clean synthesis baseline 생성.

Steps:

1. `library_setup.tcl` 작성.

```tcl
set_app_var target_library [list $env(TARGET_LIB)]
set_app_var link_library "* $env(TARGET_LIB) $env(EXTRA_LINK_LIBS)"
set_app_var search_path [concat $search_path $env(PROJECT_ROOT)/rtl/cv32e40p/rtl]
```

2. `run_dc.tcl` 작성.

Core flow:

```tcl
analyze -format sverilog -f filelists/cv32e40p_dc.f
elaborate cv32e40p_top
current_design cv32e40p_top
link
check_design
read_sdc constraints/cv32e40p_func.sdc
compile_ultra -gate_clock
write -format ddc -hierarchy -output outputs/dc/cv32e40p.ddc
write -format verilog -hierarchy -output outputs/dc/cv32e40p.vg
write_sdc outputs/dc/cv32e40p.sdc
set_svf outputs/dc/cv32e40p.svf
report_qor
report_timing
report_area
report_power
```

3. Baseline SDC.

Initial:

```tcl
create_clock -name clk_i -period 10.0 [get_ports clk_i]
set_clock_uncertainty 0.1 [get_clocks clk_i]
set_input_delay  1.0 -clock clk_i [remove_from_collection [all_inputs] [get_ports {clk_i rst_ni scan_cg_en_i}]]
set_output_delay 1.0 -clock clk_i [all_outputs]
set_case_analysis 0 [get_ports scan_cg_en_i]
```

Need review:

```text
reset false path
debug/irq static config policy
clock gating checks
```

Exit criteria:

```text
analyze/elaborate/link pass
check_design major issue 없음
netlist/ddc/svf/sdc 생성
DC timing/area/power report 생성
```

## Risk Register

### Risk 1 — Clock gate replacement fails

Cause:

```text
cv32e40p_clock_gate technology mapping mismatch
```

Mitigation:

```text
write wrapper matching exact interface
start with simple ICG mapping
verify scan_cg_en_i behavior
```

### Risk 2 — SV/FPU dependencies complicate compile

Mitigation:

```text
FPU=0 first
use cv32e40p_manifest.flist not fpu manifest
exclude simulation-only files
```

### Risk 3 — Backend collateral incomplete

Mitigation:

```text
ICC2 if full physical libs available
OpenROAD fallback if not
state limitation clearly
```

### Risk 4 — Runtime too long

Mitigation:

```text
no FPU
single baseline clock first
avoid broad sweep
backend one utilization target first
```

### Risk 5 — Existing Ibex overlap

Mitigation:

```text
Do not emphasize compile/compile_ultra or period sweep
Emphasize RTL-to-GDS, floorplan/CTS/route/post-route STA
Use CV32E40P not Ibex
```

## Final Positioning

최종 포트폴리오 포지션:

```text
AES128: 직접 설계 SoC full front-end/DFT flow
ORCA: broken flow reconstruction/debug
CV32E40P: external CPU IP RTL-to-GDS physical implementation
```

한 줄:

> CV32E40P는 기존 Ibex synthesis 실험을 대체하는 backend 중심 RISC-V 프로젝트다. 핵심은 합성 숫자 비교가 아니라 front-end netlist handoff부터 floorplan, CTS, route, post-route STA까지 연결한 RTL-to-GDS flow다.
