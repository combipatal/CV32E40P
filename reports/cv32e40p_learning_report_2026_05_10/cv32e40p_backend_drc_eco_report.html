5. Library/Pin Access Root-Cause Narrowing

최종 route DRC clean으로 가기 전 가장 중요한 관찰은 HVT OR/NOR 계열의 특정 pin access가 lower-metal DRC에 강하게 기여한다는 점이었다. 특히 OR2X1_HVT, NOR2X0/X1/X2_HVT 회피는 DRC를 크게 줄였다.

표 6. HVT OR/NOR/MUX contributor narrowing

여기서 중요한 실무적 판단은 “cell type 전체 ban”이 항상 좋은 fix가 아니라는 점이다. broad dont_use는 logic restructuring, area/timing 변화, 새로운 pin access 문제를 만들 수 있다. 그래서 이후 방향은 targeted ECO로 이동했다.

6. DRC67에서 Route DRC Clean까지

13. 실무 관점과 Claim Boundary

합성 문제가 아니라 route DRC 문제일 때

합성은 R2N/N2N/STA가 clean이어도 backend에서 DRC가 남을 수 있다. 이때 합성 script만 계속 바꾸는 것은 위험하다. 먼저 DRC class와 physical context를 보고 library/pin access/route option/PG obstruction/placement density 중 어디에 가까운지 분리해야 한다.

PD/CAD/library issue 분리 기준

  • 모든 DRC가 lower-metal과 specific ref/pin에 몰리면 library abstract/pin access 가능성이 높다.
  • PG stripe 주변에서 DRC가 바뀌면 PD floorplan/PG obstruction 영향이 있다.
  • route option만 바꿔도 class가 그대로면 route knob만으로는 root fix가 아니다.
  • LEF/GDS/NDM이 서로 다르면 CAD/library data quality issue로 분리해야 한다.

Broad dont_use vs Targeted ECO

Broad dont_use는 해당 cell type을 전체 합성에서 제거하기 때문에 구조가 크게 바뀐다. DRC가 줄 수도 있지만 timing/area/routing이 더 나빠질 수 있다. Targeted ECO는 문제 instance만 바꾸므로 변화량이 작고 설명 가능하다. 이 프로젝트에서는 broad NOR2X4_HVT ban이 481 DRC로 악화됐고, targeted NOR2X4->NOR2X2는 67까지 개선했다.

왜 full signoff라고 말하면 안 되는가

이 결과는 ICC2/PrimeTime/Formality 기반 educational FE-to-BE closure다. 하지만 foundry deck 기반 signoff DRC, LVS, antenna, IR/EM, metal fill, signoff STA methodology가 포함되지 않았으므로 tapeout-ready GDS라고 부르지 않는다.

15. 최종 결론

이 backend flow는 initial route DRC 408에서 시작해, 단순 density/route repair/route option으로 해결되지 않는 lower-metal pin-access/NDM/PG interaction 문제를 단계적으로 좁혔다. 최종적으로 route DRC clean, MCMM STA/electrical closure, Formality N2N, GDS export까지 이어졌다.

학습 가치의 핵심은 최종 숫자뿐 아니라 reject 이유를 기록했다는 점이다. 실무에서도 DRC closure는 “한 옵션을 맞히는 게임”이 아니라, report를 구조화하고 failure class를 줄이며, timing/equivalence/signoff boundary를 동시에 관리하는 과정이다.

최종 표현은 educational final-candidate GDS가 맞다. tapeout-ready, foundry signoff clean, production signoff complete라고 쓰지 않는다.
